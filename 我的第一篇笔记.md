# Obsidian + GitHub 知识库与博客构建指南

本指南将带你完成从本地写作到云端托管的完整闭环。请严格按照顺序执行。

## 第一阶段：建立云端与本地的连接（只需做一次）

### 1. 在 GitHub 创建云端仓库 (Repository)

仓库（Repo）就是你在云端的文件夹。

1. 登录 [GitHub.com](https://github.com/ "null")。
    
2. 点击右上角的 `+` 号，选择 **New repository**。
    
3. **Repository name**: 输入一个名字，比如 `my-knowledge-base` 或 `blog-source`（不要有空格，用连字符）。
    
4. **Visibility**: 如果你想以后做成公开博客，选择 **Public**；如果只想做私人云盘，选择 **Private**。
    
5. 勾选 **Add a README file**（这很重要，用于初始化仓库）。
    
6. 点击底部的 **Create repository**。
    

### 2. 用 GitHub Desktop 将仓库克隆 (Clone) 到本地

“克隆”的意思是把云端的文件夹下载到你的电脑上，并建立同步绑定。

1. 打开 **GitHub Desktop** 软件，并登录你的 GitHub 账号。
    
2. 点击左上角 `File` -> `Clone repository...`。
    
3. 在弹出的窗口中，选择 `GitHub.com` 标签页，你应该能看到你刚刚创建的仓库（例如 `my-knowledge-base`）。
    
4. 选中它，在下方的 `Local Path` 选择一个你电脑上的本地路径（例如 `D:\Documents\my-knowledge-base`）。**请记住这个路径。**
    
5. 点击 **Clone**。现在你的电脑上就有了一个与云端绑定的本地文件夹。
    

### 3. 将 Obsidian 接入该本地文件夹

1. 打开 **Obsidian**。
    
2. 在启动界面选择 **"打开文件夹作为仓库" (Open folder as vault)**。
    
3. 浏览并选择你刚才在 GitHub Desktop 中设置的那个 Local Path 文件夹（`my-knowledge-base`）。
    
4. 点击打开。此时，你在 Obsidian 里看到的 `README.md`，就是你在 GitHub 上创建的那个。
    

## 第二阶段：日常工作流 (The Daily Loop)

建立好连接后，你以后的日常使用只遵循以下三步逻辑：

### 1. 创作 (在 Obsidian 中)

1. 在 Obsidian 中新建一个文件，命名为 `我的第一篇笔记`。
    
2. 随便写点内容，例如：“测试 Obsidian 与 GitHub 的联动”。
    
3. Obsidian 会自动实时保存这个文件到你的本地硬盘。
    

### 2. 确认与打包 (在 GitHub Desktop 中)

文本虽然在本地保存了，但云端还不知道。

1. 打开 **GitHub Desktop**。
    
2. 软件会自动侦测到文件的变化。你会看到左侧栏多出了 `我的第一篇笔记.md`，并标有绿色的 `+` 号。
    
3. 在左下角的 **Summary (Required)** 文本框里，写下一句简短的修改说明，例如：“新建了第一篇笔记”。（这被称为一次 **Commit**，即提交一个历史快照）。
    
4. 点击蓝色的 **Commit to main** 按钮。这代表你在本地封存了这个历史版本。
    

### 3. 推送上云 (在 GitHub Desktop 中)

1. 刚才的 Commit 只是保存在了本地的版本库。你需要把它推送到 GitHub 服务器。
    
2. 点击软件顶部栏的 **Push origin** 按钮。
    
3. 进度条走完后，去浏览器刷新你 GitHub 上的仓库页面，你会发现 `我的第一篇笔记.md` 已经安全地躺在云端了。
    

## 第三阶段：将其变为公开博客（逻辑简述）

你现在拥有了极度安全且自带历史回溯的本地知识库。若要将其变成别人可以通过网址访问的博客，其逻辑如下：

纯文本（Markdown）浏览器是无法渲染成漂亮网页的。你需要一个“翻译器”（静态网站生成器）。 GitHub 提供了一项名为 **GitHub Pages** 的服务。它可以自动抓取你仓库里的 `.md` 文件，套用一个模板，生成 `.html` 网页。

**目前最适合 Obsidian 用户的博客方案：** 我们通常不使用 GitHub 默认的底层工具，而是使用专门为 Obsidian 优化的开源框架，例如 **Quartz (Quartz 4)**。

- **它的原理是：** 你把 Obsidian 笔记通过 GitHub Desktop 推送到一个特定的 Quartz 仓库中，GitHub 后台的一台虚拟机会自动运行代码，把你的双向链接、Markdown 语法、图片全部转换成极简、美观的网页，并为你生成一个专属网址 (例如 `yourname.github.io`)。
    

当你熟练掌握了第二阶段的日常工作流（写字 -> Commit -> Push）后，我们就可以引入 Quartz 来完成最后一步的博客部署。
